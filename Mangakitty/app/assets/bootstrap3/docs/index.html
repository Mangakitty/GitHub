Introduction

Plan is a skin for Bootstrap 3, wich means that can be used with any site built with the Bootstrap framework. With easy installation and customization thanks to the LESS files that are included. LESS is a tool to quickly declare CSS rules and after compile the .less files, get the real .css file ready to include in your site.

Plan Skin has a flat style. There's no shadows or gradients applied to the components. It also include an extended set of UI Elements based on the same style and compatibles with the standard Bootstrap Elements. See section Files to know how to include and use this new UI Elements.

Each components usage is described in the file plan.html

JS

Under the js folder you will find the Javascript files required for features

  • jquery.js: A library required for Bootstrap Javascript
  • bootstrap.js: This files contains the necessary Bootstrap Javascript for the standard components. The minified version of this files is recommended to use.
  • html5shiv.js: This files is used to add support for HTML5 tags in old browsers
  • plan.ui.js: This files contains the JavaScript required to make all the extended UI elements works correctly
  • video.js: The Video.JS main file. Required if you want to use HTML5 video in your page
  • responsive.video.js: This file is required to make your Video.JS videos responsive
  • jquery.placeholder.js: This plugin allows to use text placeholders in browsers that are not supported.
  • jquery.gmap.min.js: Google map v3 jQuery plugin
  • respond.min.js: support for media queries in older browsers (ie8)
  • application.js: This files is used for demo.

Font

This folder includes all the font files required by FontAwesome.

Img

This contains the image used in demo.

Less

This folder contains the LESS files which are the CSS source for the skin. To customize this skin you can edit the CSS files included or edit the LESS files and then compiles them into CSS. See here to learn more about LESS.

Inside this folder there are two folders

  • fontawesome: includes the FontAwesome LESS core files
  • library: include the Plan Skin LESS files for the extended UI elements.

4. Usage

CSS installation

Include the next html code in the head section of your page

  <!-- Bootstrap -->
  <link href="assets/css/bootstrap.min.css" rel="stylesheet">
  <!-- Plan UI -->
  <link href="assets/css/plan.min.css" rel="stylesheet">
  <!-- Video JS -->
  <link href="assets/css/video-js.min.css" rel="stylesheet">
  <!-- Bootstrap Select -->
  <link rel="stylesheet" href="assets/css/bootstrap-select.min.css">

Make sure to fix the path to the stylesheet if your project uses some different.

Plugin installation

Add the next scripts at the bottom of your page. Just before the </body>

  <!-- Main Scripts-->
  <script src="assets/js/jquery.js"></script>
  <script src="assets/js/bootstrap.min.js"></script>
  <!-- Video JS -->
  <script src="assets/js/video.js"></script>
  <script src="assets/js/responsive.video.js"></script>
  <!-- Bootstrap Select -->
  <script src="assets/js/bootstrap-select.min.js"></script>
  <!-- Plan UI elements -->
  <script src="assets/js/plan.ui.js"></script>

Make sure to fix the path to the script if your project uses some different.

Plan Inputs Javascript

To use the custom inputs included with this skin you need to init them via Javascript. Just use the following code:

  $(document).ready(function(){

    PlanInput.init();

  })
This function will init all components, but there are other functions that can use separately to init particular elements, for example if you are creating element loaded via ajax calls.
Some functions are: 
  PlanInput.initCheckbox();
  PlanInput.initRadio();
  PlanInput.initInputGroupCheckbox();
  PlanInput.initInputGroupRadio();

Skin installation

This section cover the skin usage, assuming that you are using the same folder structure as in the downloaded package.
To use the skin included with the package you have two alternatives

  1. Copy the files bootstrap.[theme].min.css* and plan.[theme].min.css* to the assets/css folder and link them from your pages
    Your link tags should look like:
      <!-- Example with belize skin -->
  <link href="assets/css/bootstrap.belize.min.css" rel="stylesheet">
  <link href="assets/css/plan.belize.min.css" rel="stylesheet">
  2. Rename the files bootstrap.[theme].min.css* to bootstrap.min.css and plan.[theme].min.js* to plan.min.js. Then copy them and replace the original version located in folder the assets/css. You dont need to edit html files.

*Where [theme] must be the skin name (belizehole, greensea, etc)

Font installation

If you want to include the font used in demo pages, add the following code between the <head> of your pages

  <link href='http://fonts.googleapis.com/css?family=Lato:300,700,300italic' rel='stylesheet' type='text/css'>

Compiling Source

Node.js is a platform built on Chrome's JavaScript runtime for easily building fast, scalable network applications.

Gulp is a task manager, you can define different tasks to run certain commands. Those commands does the compilation job for JADE and LESS, and concats the JS files.

The package includes under the root the file gulpfile.js and package.json to install the required componets to compile the source files.

It's not necessary that you use gulp, you can choose to work with your favorite tool to compiles LESS, JADE and concat JS file (e.g. codekit, prepros)

The following steps are intended to be an orientation guide, if you are not experienced with this you will need to learn more about it from Google :)

  1. To install node, go to http://nodejs.org/ which comes with npm (node package manager)

  2. After that, open a terminal and go to the skin root folder and run the command npm install. This command will install gulp and all depencies.

  3. Then run gulp to start the task manager

If everything goes fine, you should see the messages in the terminal telling you that most of the task are done ok. The task includes a watch to compile automatically all files when they change and also refresh the browser (requires LiveReload extension for chrome).

5. UI Kit

Using the Bootstrap 3 version of this skin you get a set of UI components created just with the features that this skin includes.

There is no special files to include or customize. The UI is based on cards concept, a design trend that divides the content into individual components. The concept is to include just the necessary information into single pieces instead of showing many pages. This concept is ideal for mobile websites and, since Bootstrap 3 is now mobile first, everything fits to provide the best user experience.

Extra files used in the UI

This is not a contradiction. As said before, there is no extra files, but according to what you need to show, you may need some extra plugins like gMap for google maps or the file upload plugin for Bootstrap.

Used files are included in the skin js folder

Panels

Bootstrap 3 panel components has been customized to match perfectly the UI Kit elements. Learn from the Bootstrap 3 website more about panels. Anyway, the markup in file uikit.html is well organized with single comments so you can copy/paste directly into your site.
